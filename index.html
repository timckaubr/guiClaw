Installation

Step 1: Installation and Uninstallation of OpenClaw in macOS

Follow these steps to install OpenClaw on macOS:

1. Install Homebrew

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Homebrew is the package manager for macOS.

2. Install Node.js

brew install node

Node.js 22+ is required for OpenClaw gateway.

3. Install OpenClaw

curl -fsSL https://openclaw.ai/install.sh | bash

This will install OpenClaw CLI and set up the gateway service.

4. Setup OpenClaw

openclaw onboard

This will run the onboarding wizard to configure OpenClaw.

Uninstallation (if needed)

openclaw uninstall --all --yes --non-interactive

This will completely remove OpenClaw from your system.

Click to check if OpenClaw is installed correctly.

Step 2: Setup of the OpenClaw Environment

Configure the OpenClaw environment for guiClaw to work properly:

2.1 Enable HTTP API Endpoints

openclaw config set gateway.http.endpoints.responses.enabled true

This is the critical step for guiClaw to work properly.

2.2 Add Trusted Proxy

openclaw config set gateway.trustedProxies '["127.0.0.1:8000", "localhost:8000"]'

This allows guiClaw to communicate with OpenClaw securely.

2.3 Restart the Gateway

openclaw gateway restart

Restart to apply the configuration changes.

Click to verify the gateway is running correctly.

Step 3: Update guiClaw to the Correct OpenClaw Gateway Token

Get your OpenClaw gateway token and update guiClaw:

3.1 Get Your Gateway Token

cat ~/.openclaw/openclaw.json | jq -r '.gateway.auth.token'

Copy the token value for the next step.

3.2 Show the Current Gateway Token

Your current OpenClaw gateway token:

Loading token...

3.3 Update Token in Index File

Update the token value by editing the index.html file in guiClaw:

const AUTH_TOKEN = 'YOUR_TOKEN_HERE';

Search for const AUTH_TOKEN in ~/Desktop/guiClaw/index.html and replace the value.

3.4 Install Dependencies

cd ~/Desktop/guiClaw && npm install busboy

Install the busboy package for file uploads.

Step 4: Refresh guiClaw

Stop the current guiClaw server and run the start script:

4.1 Stop the Current guiClaw Server

pkill -f "node proxy.js"

4.2 Run the Start Script

cd ~/Desktop/guiClaw && ./start.sh

This will start guiClaw and open your browser automatically.

4.3 Access guiClaw

Open your browser and navigate to:

http://localhost:8000

Click to stop and restart guiClaw with the updated token.

Step 1: Link WhatsApp

Link your WhatsApp account to OpenClaw:

1.1 Link WhatsApp Account

openclaw channels login --channel whatsapp

This will display a QR code for you to scan with your phone.

1.2 Scan QR Code

After running the command above:

  1. Open WhatsApp on your phone
  2. Go to Settings ‚Üí Linked Devices ‚Üí Link a Device
  3. Scan the QR code displayed in your terminal

If using pairing mode, approve the pairing request:

2.1 List Pairing Requests

View pending pairing requests.

2.2 Approve Pairing Request

openclaw pairing approve whatsapp <CODE>

Replace <CODE> with the actual pairing code.

Manage which numbers can contact OpenClaw via WhatsApp:

Skill Locations & Precedence

Skills are loaded from three locations:

  1. Workspace skills: <workspace>/skills (highest precedence)
  2. Managed/local skills: ~/.openclaw/skills (shared across agents)
  3. Bundled skills: Shipped with OpenClaw install (lowest precedence)

Precedence: Workspace ‚Üí Managed/local ‚Üí Bundled

Per-agent vs shared: Workspace skills are per-agent, managed/local skills are shared across all agents on the same machine.
